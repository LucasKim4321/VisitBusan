📌 **요구사항**

**고객**

- 고객은 웹사이트에 등록 및 로그인하여 개인 계정을 생성할 수 있어야 합니다.
- 회원의 경우 “마이페이지”와 “여행공유 게시판 글 등록”을 할수있는 권한이 있습니다.

**관리자**

- 회원정보 등록 시 등록정보를 기준으로 중복회원이 발생하지 않도록 하고 운영 담당자가 확인하고 관리할 수 있도록 구현 되어야 한다.
- 관리자는 회원탈퇴에 대한 이력을 검색할 수 있어야 하고 관련정보(탈퇴 시간 등)을 포함되어야 한다.
- 사용자의 로그인, 로그아웃, 페이지 방문 기록 등의 활동을 로그로 저장하고 관리자가 조회할 수 있도록 한다

**시스탬**

- SNS 유입정보는 기간별 로그가 제공되어야 합니다.
- 기본적인 회원가입 절차를 준수한다(본인인증, 약관동의, 개인정보보호 동의, 회원정보입력 등)
- I-PIN, 휴대폰인증, SNS인증 등의 본인확인수단을 제공하여 이용자의 편의를 높여야 한다.
- 회원가입 시 비밀번호는 발주처의 비밀번호 규칙을 안내하고 이를 준수하고 비밀번호 입력시 회원에게 위험/양호/안전 등의 수준을 실시간으로 보여주고, 안전하지 않은 비밀번호에 대해 회원가입을 제한한다.
- 회원대상으로 개인정보이용현황을 정기적으로 발송할 수 있어야 한다.
- 회원가입, 로그인, 로그아웃 버튼이 명확하게 보여져야 하며, 로그인 후 로그인 상태로 전환되었음을 명확하게 알 수 있도록 해야 한다.

📌 **기능 명세화**

- **설명**
    
    사용자의 이름, 이메일, 비밀번호를 제공하여 계정 등록
    
- **사용자 스토리**
    - 메인페이지 → 회원가입 → 필수 정보 입력 → 가입하기 → 웹사이트
    - 메인페이지 → 로그인 → sns로 로그인하기 → 웹사이트
    - 메인페이지 → 로그인 → 로그인하기 → 웹사이트
    
    **비밀번호 재설정**
    
    - 메인페이지 → 로그인 → 비밀번호 찾기 → 이메일 입력 → 이메일로 전송된 링크 클릭 → 새 비밀번호 입력 → 비밀번호 재설정 완료 → 웹사이트
    
    **계정 잠금 해제**
    
    - 메인페이지 → 로그인 → 계정 잠금 해제 요청 → 이메일 입력 → 이메일로 전송된 링크 클릭 → 계정 잠금 해제 완료 → 웹사이트
</aside>

📌 **비기능 요구사항**

- **보안 요구사항**
    
    모든 사용자 비밀번호는 암호화되어 저장되어야 합니다.(전송 중인 데이터뿐만 아니라 저장된 데이터도 암호화)모든 사용자 비밀번호는 암호화되어 저장되어야 합니다.(전송 중인 데이터뿐만 아니라 저장된 데이터도 암호화)
    
    로그인 시도가 여러 번 실패 계정이 잠금 상태로 전환되어야 합니다.
    
    일정 시간 동안 활동이 없을 경우 자동으로 로그아웃 처리합니다.(여러 번 로그인 시도 실패로 계정이 잠긴 경우, 이메일을 통해 계정 잠금을 해제할 수 있는 기능을 제공)
    
    비밀번호는 최소 8자 이상, 숫자, 대문자, 소문자, 특수문자를 포함해야 합니다.
    
- **성능 요구사항**
    
    웹사이트 평균 로딩 시간은 3초 이내
